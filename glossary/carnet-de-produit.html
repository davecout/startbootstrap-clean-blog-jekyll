Un carnet de produit est un document ou une collection de documents qui décrivent les exigences, les fonctionnalités et les objectifs d'un produit. Il peut inclure des informations telles que les objectifs de marché, les personas utilisateurs, les scénarios d'utilisation, les exigences fonctionnelles et non fonctionnelles, les critères de qualité, les plannings, les budgets et les risques associés au produit. Le carnet de produit est généralement utilisé pour planifier et gérer le développement d'un produit, et il est utilisé pour planifier les itérations. Il est généralement maintenu et mis à jour par les propriétaires de produit ou les responsables de produits. Il est utilisé pour communiquer les objectifs et les exigences du produit aux différentes parties prenantes, pour s'assurer que tout le monde est sur la même longueur d'onde et pour s'assurer que tout le monde travaille dans la même direction.

Le carnet de produit contient:

  • Les épopées,
  • Les fonctionnalités,
  • Les récits utilisateur,
  • Les tâches, et
  • Les anomalies.
